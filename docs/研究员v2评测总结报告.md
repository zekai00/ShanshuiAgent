## 运行环境说明

本次检索服务启动成功，但 Neo4j 未连接，图谱检索自动降级为空结果。因此本次是 Milvus/BGE/ColBERT/Reranker 的无图谱结果，不代表完整 Graph RAG 上限。

回答评测调用当前 `researcher_node`，被测回答模型是 `deepseek-v4-flash`，评判使用 LLM-as-judge。结论用于工程决策，不能当作论文级人工金标结论。

## 主要发现

1. 正常问答能力不差。v2 主集回答通过率 89.00%，说明当前 DeepSeek Researcher 在有检索证据时能稳定完成多数事实、概念、比较和创作转写题。
2. 错误前提仍是最明显风险。主集 `evidence_missing` 通过 20/25，独立错误前提专项也是 20/25；失败多是“先否定，但随后展开类比或顺着错误前提讲太多”，例如 Midjourney、VR、空调制冷等问题。
3. 检索的薄弱点是指定文献召回。概念解释题来源命中只有 22/40，常返回通用概念论文，而不是题目指定 PDF。
4. 画师创作转写不够稳。`artist_prompt` 通过 21/25，失败多来自回答没有足够严格地把“考据依据”和“可执行创作要点”分开。
5. v2 评测集需要人工清洗。自动生成题中有少量标题截断、主题过泛、比较对象不自然的问题，会放大自动裁判的不确定性。

## 对训练策略的判断

现在不建议直接上 DPO/PPO/GRPO。

更合理的顺序是：

1. 先清洗 v2 评测集，形成 `reviewed` 版本，避免训练和评测被坏题干误导。
2. 修检索标题锚定和 Neo4j 图谱可用性，尤其是“根据《某文献》”的问题。
3. 调 Researcher prompt，让错误前提题强制采用“先纠错、再限界、少类比”的结构。
4. 如果要训练小模型，第一阶段仍然是 SFT：训练“检索证据 -> 有边界回答 -> 规范引用 -> 错误前提拒答”的稳定格式。
5. DPO 只适合 SFT 之后，用 chosen/rejected 对修正偏好，例如拒绝展开不必要类比、减少脱离证据发挥、提高引用严谨性。

## 下一步建议

短期先做三件事：

1. 人工复核 v2 中失败样本和自动题干异常样本，输出 `data/eval/test_researcher_v2_reviewed.jsonl`。
2. 修检索：标题字段加权、过滤 `paper*.pdf` 噪声来源、恢复或禁用 Neo4j 图谱路径。
3. 修改 Researcher 提示词：错误前提题回答不应展开“虽然古人没有现代技术，但两者有相似机制”的长篇类比，除非用户明确要求跨时代比较。

完成这三步后，再跑一次 v2 reviewed 评测，决定是否继续 SFT。
