# RAG 证据链路与权威索引重构报告

生成时间：2026-05-31 03:39 CST

## 本轮目标

将 RAG 从“旧 Milvus 结果迁移出的证据补丁”推进为“文献清单 → PDF 文本审计 → 页级文本 → chunk 证据 → Milvus 索引”的可重复链路，并默认让在线检索读取新的权威语料索引。

## 已完成改动

1. 新增 `scripts/retrieval/build_authority_evidence_store.py`
   - 输入：`data/metadata/文献级标注清单.jsonl` 与 `data/metadata/PDF文本可抽取性审计.jsonl`。
   - 只导入 `text_ready` 文献；`needs_ocr`、`low_text`、缺失或解析失败文献进入 `skipped_documents.jsonl`。
   - 输出页级和 chunk 级证据，保留 `doc_id`、`source_file`、`title`、`author`、`page_start/page_end`、权威等级、类别、主题、朝代、流派、人物、来源 URL 等结构字段。

2. 生成新的 canonical evidence store
   - 输出目录：`data/processed/authority_evidence/`
   - 文献：135 篇
   - 页：3617 页
   - chunk：7956 个
   - 跳过：29 篇，其中 `needs_ocr` 28 篇，`low_text` 1 篇
   - 每个 chunk 都有稳定 `chunk_id` 与可写入 Milvus 的稳定整数 `legacy_milvus_id`。

3. 新增 `scripts/retrieval/build_milvus_from_evidence_store.py`
   - 从 `authority_evidence/chunks.jsonl` 构建 Milvus Lite 索引。
   - 本轮构建 dense+sparse 双路索引，未生成 ColBERT 张量，避免额外占用约十几 GB 磁盘。
   - 新索引：`data/vector_store/milvus_authority.db`
   - 插入 chunk：7956 个
   - 构建耗时：110.72 秒
   - 索引大小：约 64 MB

4. 在线检索默认切到新链路
   - 新增配置：`CL_RETRIEVAL_MILVUS_DB_PATH`、`CL_RETRIEVAL_COLBERT_TENSORS_PATH`、`CL_RETRIEVAL_EVIDENCE_DIR`。
   - `src/retrieval/online_retrieval.py` 改为读取 retrieval 专用 Milvus 与 evidence store。
   - 旧 `data/processed/documents/` 与旧 `milvus_landscape.db` 保留，不再作为默认在线检索链路。

## 验证结果

1. 编译检查通过：
   - `python -m py_compile $(find src scripts ui -name '*.py' -not -path '*/__pycache__/*')`

2. Milvus 结构检查通过：
   - 能从 `milvus_authority.db` 查询出包含 `id/source_file/page_start/chunk_id` 的记录。

3. 端到端 smoke test 通过：
   - 命令：`CL_MODEL_DEVICE=cuda:1 python scripts/retrieval/smoke_test_retrieval.py --query '郭熙的三远法是什么？' --top-k 5 --final-k 3`
   - 命中结果：3 条
   - Top 结果均来自 `谈郭熙“三远法”对山水画创作的影响_NormalPdf.pdf`，并返回精确页码与 chunk_id。

## 当前链路形态

当前检索返回的不只是向量命中结果，而是：

- Milvus 中的向量候选：`id`、dense/sparse 向量、`contextual_chunk`、来源文件、朝代、人物、主题等动态字段。
- evidence store 中的结构证据：`chunk_id`、`doc_id`、`source_file`、`title`、`page_start/page_end`、原文 chunk、上下文前缀、权威等级、来源 URL、分类维度。
- 回答模型拿到的是“来源文献 + 页码 + 证据块 ID + 原文内容”，可以据此生成可核验回答。

## 已知限制

- 28 篇扫描型 PDF 仍需 OCR 后才能进入文本索引。
- 新索引暂未启用 ColBERT 多向量召回；当前为 dense+sparse+Neo4j 图谱召回。考虑磁盘只剩约 36 GB，本轮先不生成大型 ColBERT 文件是更稳妥的选择。
- 首次 smoke test 包含 BGE 与 reranker 初始化，耗时约 33 秒；常驻检索服务会在 lifespan 阶段预热，后续请求会更稳定。
