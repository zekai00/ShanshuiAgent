# Agent 系统改造与节点说明报告

生成时间：2026-06-01 13:09 CST

## 本次改造结论

本次把前端主对话链路从单纯 `/api/chat/stream` RAG 问答，升级为 `/api/agent/stream` 受控 Agent 流程。

新的主链路不是让大模型自由决定一切，而是用明确节点和结构化状态控制执行顺序：

```text
intake
 → planner
 → researcher
 → verifier
 → research_synthesizer
 → prompt_designer
 → image_generator
 → image_critic
 → final_writer
 → memory_writer
```

不是所有问题都会跑完全部节点。闲聊只走直接回复；普通研究问题只走检索、核验、回答；研究创作问题才走完整的“检索证据 → 形成卷宗 → 生成 Prompt → 调用 ComfyUI → 检查图像 → 交付”链路。

## 已落地的核心能力

### 1. 新增 Agent 流式接口

新增接口：

```text
POST /api/agent/stream
```

返回 NDJSON 流式事件，包括：

- `phase`：当前阶段。
- `plan`：本轮执行计划。
- `node`：节点状态，包含 `running`、`done`。
- `evidence`：RAG 检索证据。
- `brief`：研究卷宗。
- `image_spec`：图像生成 Prompt 和尺寸。
- `image`：ComfyUI 图像生成结果。
- `image_critic`：图像检查结果。
- `delta`：最终回答文本流。
- `done`：本轮结束。

旧接口 `/api/chat/stream` 保留，方便回退和评测。

### 2. 前端主对话切换到 Agent

前端发送问题时现在调用 `/api/agent/stream`。

新增显示：

- Agent 工作流时间线。
- 每个节点的执行状态。
- 图像 Prompt 折叠面板。
- 图像生成结果或失败原因。
- 图片下载入口。

### 3. 图像生成链路正式纳入 Agent

系统现在会把 ComfyUI 作为 `image_generator` 节点工具使用。

当前图像工具行为：

- 检查 ComfyUI 是否在线。
- 读取 `workflows/flux1_krea_dev-api.json`。
- 写入 positive prompt、尺寸、seed。
- 调用 ComfyUI `/prompt`。
- 轮询 `/history/{prompt_id}`。
- 下载生成图像到 `generated_images/`。
- 通过 `/generated-images/{filename}` 提供前端访问。

如果 ComfyUI 未启动，系统不会伪造图像，而是返回失败状态，并继续交付研究卷宗和可用 Prompt。

## 节点说明

### `intake`：任务理解节点

职责：

- 判断用户问题类型。
- 识别是否属于中国山水画研究范围。
- 识别是否需要生成图像。
- 抽取朝代、流派、技法、画家等实体。
- 检查明显错误前提。

输入：

- 用户原始问题。
- 当前路由规则。

输出示例：

```json
{
  "task_type": "research_then_image",
  "needs_retrieval": true,
  "needs_image": true,
  "entities": {
    "dynasties": ["明"],
    "schools": ["吴门"],
    "techniques": [],
    "artists": []
  }
}
```

失败处理：

- 闲聊问题不进入检索。
- 非山水画图像请求会被拦截。
- 明显时代错置问题会转入前提纠偏。

### `planner`：计划制定节点

职责：

- 根据 `intake` 的任务类型生成执行计划。
- 决定本轮是否需要检索、是否需要生图、是否需要记忆写入。

典型计划：

```text
researcher → verifier → research_synthesizer → prompt_designer → image_generator → image_critic → final_writer → memory_writer
```

失败处理：

- 如果任务不需要检索，只生成短链路。
- 如果任务越界，只安排 `verifier` 和 `final_writer`。

### `researcher`：文献检索节点

职责：

- 调用当前 RAG 检索系统。
- 使用 Milvus/evidence store 检索候选证据。
- 使用 reranker 重排。
- 返回带来源、页码、PDF 链接、页图链接的证据结构。

输入：

- 用户问题。
- `top_k`、`final_k`。

输出：

- `evidence` 列表。

失败处理：

- 检索失败会转为 Agent 错误事件。
- 检索结果相关性不足时交给 `verifier` 拦截。

### `verifier`：证据核验节点

职责：

- 判断证据是否足够相关。
- 判断错误前提是否需要拒答或纠偏。
- 判断是否可以进入研究回答或图像创作。

当前核验逻辑：

- 若最高 rerank 分数低于阈值，进入证据不足回复。
- 若问题把古代画家、画派和现代生成技术混为历史事实，直接纠偏。
- 若问题涉及“梵高是否直接学习元四家”等直接影响关系，证据不足时不默认成立。

输出示例：

```json
{
  "verdict": "ok",
  "can_continue": true,
  "reason": "证据可用于下一步。"
}
```

### `research_synthesizer`：研究卷宗节点

职责：

- 把检索证据整理成下游画师可用的研究卷宗。
- 提炼画史、构图、笔墨、设色、视觉约束。
- 保留证据编号，避免脱离来源。

输出示例：

```json
{
  "topic": "明代吴门画派风格",
  "key_points": ["[1] 吴门画派重视江南实景和文人经验..."],
  "visual_constraints": ["江南实景气息", "文人清雅", "平远空间"],
  "citations": [1, 2, 3]
}
```

失败处理：

- 如果 LLM 不可用，使用证据摘要和实体规则生成 fallback 卷宗。

### `prompt_designer`：图像提示词设计节点

职责：

- 把研究卷宗转成 ComfyUI/Flux 可用的图像参数。
- 输出英文 positive prompt。
- 输出 negative prompt。
- 决定画幅尺寸和格式。

当前规则：

- 长卷/横幅：`1536 x 768`
- 立轴/竖幅：`768 x 1536`
- 默认斗方：`1024 x 1024`

输出示例：

```json
{
  "format": "horizontal_scroll",
  "width": 1536,
  "height": 768,
  "positive_prompt": "Chinese shanshui landscape painting, Ming dynasty Wu School style...",
  "negative_prompt": "photorealistic, oil painting, modern buildings..."
}
```

失败处理：

- 如果 LLM 不可用，使用研究卷宗中的视觉约束生成规则版 Prompt。

### `image_generator`：图像生成节点

职责：

- 调用 ComfyUI 工作流。
- 写入 prompt、尺寸、seed。
- 下载生成结果。
- 返回前端可访问 URL。

成功输出：

```json
{
  "status": "success",
  "url": "/generated-images/xxx.png",
  "seed": 123456,
  "workflow": "workflows/flux1_krea_dev-api.json"
}
```

失败输出：

```json
{
  "status": "failed",
  "error_type": "comfyui_offline",
  "message": "ComfyUI 当前不可用"
}
```

失败处理：

- 不伪造图像。
- 保留 Prompt 和研究卷宗。
- 由 `final_writer` 明确说明当前未实际出图。

### `image_critic`：图像检查节点

职责：

- 检查图像是否真的生成。
- 检查文件是否存在且非空。
- 检查 Prompt 是否包含基本山水画约束。

当前是轻量检查，不做视觉模型判读。

后续可扩展：

- 接入视觉模型判断是否像山水画。
- 检查是否出现现代建筑、照片感、油画感。
- 自动修订 Prompt 并重试。

### `final_writer`：最终交付节点

职责：

- 整理最终回答。
- 对研究问答输出带引用的回答。
- 对研究创作任务输出图像、Prompt、研究依据和来源。
- 对失败情况明确说明，而不是编造结果。

输出内容包括：

- 研究依据要点。
- 创作约束。
- 图像状态。
- positive prompt。
- negative prompt。
- 来源。

### `memory_writer`：记忆写入节点

职责：

- 只记录用户明确表达的偏好。
- 不记录普通问答流水账。

当前规则：

- 识别“我喜欢……”。
- 识别“偏好……”。
- 识别“以后不要……”。
- 识别“不喜欢……”。

默认前端没有传真实用户 ID，因此一般是 guest 模式，不会持久写入。

## 当前限制

- `image_critic` 目前只做文件级和 Prompt 级检查，还没有接入视觉模型。
- ComfyUI 必须单独启动，否则图像节点会返回 `comfyui_offline`。
- 当前 Agent 是受控状态机，不是完全自由工具调用 Agent；这是为了降低幻觉和不可控工具调用风险。
- 旧 `src/agent/graph.py` LangGraph 原型仍保留，但现代前端主链路已经切到新的受控 Agent 接口。

## 下一步建议

1. 给 `image_critic` 接入轻量视觉模型，检查生成图是否符合山水画、朝代和流派约束。
2. 给 `prompt_designer` 增加可编辑 Prompt 面板，让用户出图前调整。
3. 增加“重新生成”按钮，复用同一研究卷宗但更换 seed 或修订 Prompt。
4. 将 `memory_writer` 接入前端用户 ID，让偏好可以跨会话生效。
5. 建立 Agent 评测集，分别评测路由、证据、Prompt、图像生成失败处理和最终交付。
