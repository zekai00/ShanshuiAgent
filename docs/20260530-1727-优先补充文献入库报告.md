## 对 RAG 导入的影响

现在不应把 `高权重导入队列.jsonl` 全量无条件导入 Milvus。更合理的导入策略是：

1. 先导入 `PDF文本可抽取性审计.jsonl` 中 `extractability = text_ready` 的文档。
2. 对 `需OCR文献清单.jsonl` 中的高权重文档先做 OCR。
3. OCR 后重新生成 chunk，并在 chunk metadata 中保留 `doc_id`、页码、来源 URL、权威等级、OCR 状态。
4. 再把 OCR 后文本纳入 Milvus，避免出现“高权重来源存在，但检索不到正文”的假象。

## 下一步建议

下一步优先做 OCR 与导入过滤，而不是继续盲目扩文献：

1. 给 `需OCR文献清单.jsonl` 中高权重文档跑 OCR。
2. 修改 RAG ingest，只导入 `text_ready` 或 `ocr_ready` 文档。
3. 重建 Milvus 索引，并在检索结果里显示 `doc_id / title / page / source_url / extractability`。
4. 用之前的研究员评测集重新评测一次，专门看“引用来源可核验率”和“回答是否真正使用新增原典/故宫专题证据”。
