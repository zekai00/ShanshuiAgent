# RAG 证据链路重构报告

生成时间：2026-05-30 14:14 CST

## 结论

已完成第一阶段 P0 证据链路重构。此次重构没有重新解析全部 PDF，也没有重建向量；它是在现有 Milvus 数据之上增加 canonical evidence store，并让在线检索返回结构化证据。

本阶段解决了三个核心问题：

1. Milvus 不再是唯一证据源，已落盘 `data/processed/documents`。
2. 检索结果不再只有 `contextual_chunk` 字符串，已返回 `chunk_id`、`raw_chunk_text`、`contextual_prefix`、`doc_id`、`source_file` 等结构化字段。
3. 用户问题中出现 `《文献名》` 时，检索器会解析 source constraint，并优先从指定 PDF 中取证。

## 本次新增和修改

### 新增 canonical evidence store 构建脚本

文件：

- `scripts/retrieval/build_evidence_store_from_milvus.py`

作用：

- 从现有 Milvus `landscape_rag` 集合迁移 chunk payload。
- 拆分 `【全局上下文】` 与 `【原文资料】`。
- 为每个 chunk 生成稳定 `chunk_id`。
- 为每篇 PDF 生成 `doc_id`。
- 建立标题 alias，支持后续 source-aware retrieval。
- 抽取旧 chunk 中的图片路径引用。

已生成：

- `data/processed/documents/documents.jsonl`
- `data/processed/documents/chunks.jsonl`
- `data/processed/documents/images.jsonl`
- `data/processed/documents/pages.jsonl`
- `data/processed/documents/source_aliases.json`
- `data/processed/documents/manifest.json`

### evidence store 统计

本次迁移结果：

- documents：53
- chunks：4703
- pages_with_image_refs：332
- images：708
- source_aliases：53

注意：当前页码只在旧 chunk 中出现图片路径且图片文件名包含页码时可推断。文本-only chunk 的页码在旧 Milvus 中不存在，所以不能伪造页码。

### 修改在线检索器

文件：

- `src/retrieval/online_retrieval.py`

主要变化：

- 启动时加载 `data/processed/documents/chunks.jsonl`。
- 建立 `legacy_milvus_id -> evidence chunk` 映射。
- 建立 `source_file -> legacy_milvus_id` 映射。
- 建立标题 alias 映射。
- 解析用户问题中的 `《文献名》`。
- 对指定文献题增加 source-aware lexical candidates。
- RRF 融合时提高 source-constrained candidates 权重。
- reranker 精排后优先返回指定文献中的证据。
- 返回结构化字段：
  - `chunk_id`
  - `legacy_milvus_id`
  - `doc_id`
  - `title`
  - `source_file`
  - `raw_chunk_text`
  - `contextual_prefix`
  - `retrieval_text`
  - `page_start`
  - `page_end`
  - `source_constraints`
  - `source_constraint_match`
  - `evidence_store_hit`

### 修改 Researcher 工具输出

文件：

- `src/agent/tools.py`

主要变化：

- 工具输出现在包含 `证据块ID`。
- 如果能推断页码，会输出页码。

- 保留 `来源文献: xxx.pdf` 格式，兼容现有引用抽取和评测脚本。

### 修改 Researcher prompt

文件：

- `src/agent/prompts/researcher_prompt.yaml`

主要变化：

- 明确“系统上下文只用于理解原文，不等同于原文证据”。
- 要求优先依据“内容详情”中的原文片段回答。
- 如果检索结果提供证据块 ID，应在回答中说明证据块。

## 当前实现边界

这次是“旧 Milvus 数据之上的证据链路重构”，不是完整 PDF 重解析。

已经做到：

- chunk 级稳定 ID。
- 原文片段和系统上下文分层。
- source-aware 检索约束。
- 结构化 evidence 输出。
- 检索评测可统计 evidence store 命中。

尚未做到：

- 文本 chunk 的真实 PDF 页码。
- bbox 坐标。
- section title 的真实解析。
- 基于 PDF layout 的重新切分。
- BM25/full-text 独立索引。
- citation verifier 的生成后自动纠错。
- 多模态图片向量检索。

原因：旧 Milvus 里没有保存这些字段，不能凭空生成。后续需要重新解析 PDF 才能补齐。
