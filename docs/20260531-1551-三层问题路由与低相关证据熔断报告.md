# 三层问题路由与低相关证据熔断报告

生成时间：2026-05-31 15:51 CST

## 目标

将此前的纯关键词分流升级为三层路由，避免闲聊、通用问题或低相关检索结果被强行包装成山水画文献证据。

## 已完成实现

### 1. 第一层：规则快速判断

后端将关键词拆成两类：

- 强领域词：山水、中国画、水墨、青绿、皴法、笔墨、画论、三远、画家名、流派名、作品名等。
- 模糊领域词：宋代、元代、明代、清代、明清、绘画、美术等。

明显闲聊通过规则直接分流：

- 问候
- 天气
- “什么是爱”等通用生活/哲学问题

强领域词直接进入 RAG。模糊领域词不会直接进入 RAG，而是进入第二层。

### 2. 第二层：LLM 路由分类

对模糊问题或规则不确定的问题，后端会在有 API key 时调用轻量 LLM 分类。

分类标签限定为：

- `domain_research`：中国山水画研究问题，进入 RAG。
- `casual`：闲聊。
- `unsupported_general`：一般知识、哲学、生活问题。
- `need_clarification`：可能相关但太泛，需要用户补充山水画角度。

可通过环境变量控制：

- `CL_ROUTER_LLM_ENABLED=0` 可关闭 LLM 路由。
- `CL_ROUTER_LLM_MODEL` 可指定路由模型，默认复用当前快速回答模型。

### 3. 第三层：RAG 相关性阈值熔断

即使问题进入 RAG，后端也会检查 reranker 最高分。

- 默认阈值：`CL_RAG_MIN_RERANK_SCORE=1.0`
- 如果最高相关性低于阈值，系统不返回证据，不生成带来源回答。
- 低相关问题会提示用户补充画家、朝代、作品、流派或技法。

这一步用于处理检索器“低分也返回兜底 Top-1”的情况。

### 4. 前端状态

系统状态页新增“路由策略”，显示当前是否启用 LLM 路由，以及低相关阈值。

静态资源版本更新为 `20260531-1551`。

## 预期效果

1. “今天天气真不错”不检索山水画证据。
2. “什么是爱”不检索山水画证据。
3. “明清有什么区别”这类泛历史问题不会仅因包含“明清”就强行走 RAG。
4. “明清山水画有什么区别”仍会进入 RAG。
5. 如果检索结果相关性过低，即使进入 RAG，也不会展示牵强来源。

## 验证结果

- `python -m py_compile scripts/run_web_app.py`：通过。
- `node --check ui/modern/app.js`：通过。
- `今天天气真不错`：`mode=direct`，`route=casual/rule`，证据数 0。
- `什么是爱`：`mode=direct`，`route=casual/rule`，证据数 0。
- `明清有什么区别`：`mode=direct`，`route=need_clarification/llm`，证据数 0。
- `明清山水画有什么区别`：`mode=rag`，证据数 3。
- `富春山居图是谁画的？`：`mode=rag`，证据数 3。
- `山水画和量子力学有什么关系？`：`mode=low_relevance`，最高 rerank 低于 1.0，证据数 0。
- `/health` 已返回 `router` 字段：LLM 路由启用，路由模型为 `deepseek-v4-flash`，最低相关性阈值为 `1.0`。
