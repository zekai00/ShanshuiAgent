## 错误前提专项为什么还有来源

错误前提题分两类：

1. 纯时代错置或现代技术错置题，例如“王维是否使用 Midjourney”“荆浩是否提出 Transformer 注意力机制”。这类题通常不需要检索文献证明，`gold_sources=[]`，主要评估回答是否在开头直接纠错并拒绝编造。
2. 需要文献支持纠错的错配题，例如“倪瓒是否是清代四王之一”“元四家是否由王时敏、王鉴、王翚、王原祁组成”。这类题需要取回能说明正确归属的文献，所以会有来源约束，但来源的作用是支持纠错，不是支持错误前提。

因此，旧报告中“有金标来源题数”“金标来源命中”的说法不严谨。正确表述应为“有来源约束题数”“纠错证据来源命中”。错误前提专项最终应优先评估：是否开头纠错、是否拒绝顺着错误前提编造、是否只在有证据时引用文献。

## Neo4j 启用方式

`.env` 中已经存在：

```env
NEO4J_URI="bolt://127.0.0.1:7687"
NEO4J_USERNAME="neo4j"
NEO4J_PASSWORD=***
```

当前环境下，普通 `neo4j start` 会被命令执行环境回收，表现为日志显示 Started 但进程随后消失。因此本次使用 `setsid` 以后台 console 模式启动：

```bash
setsid bash -c 'exec neo4j-community-2026.03.1/bin/neo4j console' > neo4j-community-2026.03.1/logs/neo4j-console.out 2>&1 < /dev/null &
```

验证命令：

```bash
neo4j-community-2026.03.1/bin/neo4j status
```
