# Agent 语义路由与前端命名优化报告

生成时间：2026-06-01 13:30 CST

## 问题背景

用户反馈当前 Agent intake 仍偏规则化，存在三类问题：

- 口语化生图请求可能识别失败，例如“来搞个宋代山水图生成下看看”。
- 上下文续问可能识别失败，例如先问《富春山居图》后再问“能画一副类似的画吗？”。
- 一般中国绘画史事实问题被错误拒答，例如“清明上河图是谁画的”。
- 前端仍显示“山水画研究员”，与现在的研究创作 Agent 定位不一致。
- 前端状态条显示“最高相关性 3.43”，对普通用户没有必要。

## 已完成改动

### 1. Agent intake 改为 LLM 语义路由优先

新增 `classify_agent_intake_with_llm()`。

现在 Agent intake 会把最近对话和当前问题一起交给轻量 LLM 路由器，输出结构化 JSON：

```json
{
  "task_type": "research_then_image",
  "needs_retrieval": true,
  "needs_image": true,
  "entities": {
    "dynasties": [],
    "schools": [],
    "techniques": [],
    "artists": [],
    "works": []
  },
  "reason": "...",
  "confidence": 0.95
}
```

支持的 `task_type`：

- `direct`
- `general_art_qa`
- `research_qa`
- `research_then_image`
- `unsupported_image`
- `need_clarification`
- `unsupported_general`
- `invalid_premise`

### 2. 保留规则兜底和硬安全拦截

LLM 路由优先，但没有完全删除规则。

原因：

- LLM 可能因表达省略而保守判断。
- 明显错误前提不应该交给生成模型自由发挥。
- ComfyUI/Stable Diffusion 与古代画家混用为历史事实时，需要硬拦截。

因此现在是：

```text
硬错误前提拦截
 → LLM 语义路由
 → 上下文图像续问保护
 → 规则兜底
```

### 3. 增加上下文续问保护

新增 `history_has_landscape_context()`。

如果当前问题是图像创作意图，且最近对话中出现山水画作品、画家、画派或技法上下文，则将其视为 `research_then_image`。

这修复了：

```text
用户：《富春山居图》是谁画的？
用户：能画一副类似的画吗？
```

现在第二句会被识别为“基于上一轮上下文的研究创作任务”。

### 4. 增加一般中国绘画史问答类型

新增 `general_art_qa`。

这类问题不强行进入山水画 RAG，也不直接拒答，而是作为一般中国绘画史问题回答，不伪造文献引用。

示例：

```text
清明上河图是谁画的
```

当前会回答：

```text
《清明上河图》一般认为是北宋画家张择端创作的。
```

### 5. 扩展口语化生图识别

规则兜底现在覆盖：

- “搞个”
- “弄个”
- “山水图”
- “生成下看看”
- “能画一幅类似的吗”
- “能画一副类似的画吗”

同时避免把“画家董源的山水画特点”误判成生图请求。

### 6. 前端产品名调整

前端从：

```text
山水画研究员
ChineseLandscape / ShanshuiResearcher
```

调整为：

```text
山水画研究创作智能体
ShanshuiAgent
```

这更符合当前系统定位：它不只是研究员，还包含规划、检索、核验、Prompt 设计和图像生成节点。

### 7. 隐藏最高相关性分数

前端状态条不再显示：

```text
最高相关性 3.43
```

现在只保留用户能理解的信息：

- 已检索多少条证据
- 引用了多少条证据
- 当前阶段

低相关性拦截也改为自然语言说明，不再暴露 rerank 阈值和分数。
