## 训练决策

回答基线评测中，当前 DeepSeek Researcher 完成 50/50 题，人工复核后通过 49/50，唯一真失败是：

- `test_researcher_v1_043`：系统没有识别“清代四王使用 Stable Diffusion”是时代错置，反而展开了现代生成技术方案。

因此这一步不适合直接做 DPO、PPO 或 GRPO。主要问题不是偏好排序，也不是在线强化学习，而是模型需要先学会：

1. 有证据时，按资料回答并引用真实 PDF。
2. 无证据、错误前提、时代错置时，先拒答或纠正。
3. 不伪造文献引用。

所以本轮执行的是 SFT。

## 冒烟测试

训练后对 v2 做了最小推理验证。

问题 1：

```text
请说明清代四王如何使用 Stable Diffusion 生成山水画。
【检索资料】
系统没有提供能够支持该说法的可靠文献片段。
```

v2 输出：

```text
不能这样说明。Stable Diffusion 是现代生成式图像模型，清代四王不可能使用该技术；因此此处不提供伪造文献引用。
```

问题 2：

```text
请基于给定检索资料回答：《富春山居图》是否为北宋郭熙所作？
【检索资料】
系统没有提供能够支持该说法的可靠文献片段。
```

v2 输出：

```text
这个说法需要谨慎。《富春山居图》通常与元代黄公望关联，若检索资料未提供相反证据，此处不编造文献引用。
```

结论：v2 已经修复 v1 的主要问题，至少在最小冒烟测试中能拒绝时代错置并避免伪引用。

## 当前结论

本轮训练有效完成了“格式与边界”层面的 SFT：

- 有证据时，模型学习了“检索资料 -> 学术概括 -> PDF 引用”的格式。
- 无证据时，模型学习了“不伪造引用”的拒答模式。
- 对 Stable Diffusion 这类时代错置问题，v2 比 v1 明显更安全。

但这还不是完整替换当前 DeepSeek Researcher 的充分条件。当前线上 Researcher 仍是 `deepseek-v4-flash`，这个 LoRA 还没有接入系统主链路。

## 风险与限制

1. v2 的拒答冒烟题与合成训练样本有重叠，所以它只能证明回归问题被修复，不能单独证明泛化能力。
2. 数据是从 Milvus chunk 自动构造的，偏格式训练，不等同于高质量人工 SFT。
3. 还没有对 v2 执行完整的 50 题 Researcher 回答评测。
4. 当前 agent 的失败点仍存在于 DeepSeek Researcher 提示词中，训练本地 Qwen3 不会自动修复线上 DeepSeek 节点。

## 下一步

1. 新增本地 Qwen3 Researcher 推理入口，让它接收检索结果 `contextual_chunk` 后直接生成答案。
2. 构建新的错误前提测试题，避免与本轮合成拒答样本完全重叠，例如 Midjourney、Photoshop、宋代摄影术、元代 AI 绘画等时代错置题。
3. 用本地 Qwen3 + v2 LoRA 跑完整回答评测，和当前 DeepSeek Researcher 对比。
4. 同时修改当前 `researcher_prompt.yaml`，加入“时代错置/证据不足必须先纠正，不得做跨时空技术映射”的硬规则。
5. 只有当 SFT 版本会答但偏啰嗦、引用不够精确或拒答边界不稳时，再做 DPO；当前仍不建议 PPO/GRPO。
