## 结论

证据链路重构后的检索评测已经完成。

主集结果明显提升：在 reviewed 主集上，有金标来源的题目中，任一金标来源命中达到 188/194（96.91%），全部金标来源命中达到 180/194（92.78%）。所有返回证据块都命中了 canonical evidence store，chunk 级证据返回率为 631/631（100.00%）。

错误前提专项没有明显提升：有金标来源的题目中，任一金标来源命中为 13/23（56.52%），全部金标来源命中为 11/23（47.83%）。这说明 source-aware retrieval 对“根据《某文献》”这类显式指定来源问题效果很好，但对没有显式文献约束的错误前提题，还需要 query parser、实体纠错和 BM25/full-text 支持。

本轮评测期间 Neo4j 仍然离线，系统按设计降级为 Milvus/BGE-M3/ColBERT/reranker/source-aware evidence 检索。

## 仍然存在的问题

1. Neo4j 离线，graph retrieval 没有参与本轮评测。
2. 旧 Milvus 没有真实页码，当前不能输出可靠 page citation。
3. 还没有 BM25/full-text，专名和错误前提纠错题仍不稳。
4. comparison 题需要更强的多来源配额，不能让一个来源占满全部结果。
5. evidence_missing 题需要独立 retrieval planner，而不是普通语义检索。
6. 还没有 citation verifier，回答层仍可能引用不存在或未检索到的文献。

## 下一步建议

优先做三件事：

1. 多来源配额修复  
   comparison 题如果解析出两个文献约束，每个来源至少保留 2 个候选，再进入 evidence pack。

2. 错误前提 query parser  
   从问题里抽取正确实体和错误实体。例如“倪瓒是否是清代四王之一”应拆成“倪瓒”“清代四王”“元代/清代对照”，而不是直接检索整个错误句。

3. BM25/full-text  
   用于精确命中画家、流派、术语、文献题名。尤其是元四家、四王、斧劈皴、留白、三远法这类专名。

暂时不建议进入 SFT/DPO。当前主要收益仍来自检索链路修复，而不是模型训练。
