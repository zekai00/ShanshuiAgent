# 语料库与系统状态界面优化报告

生成时间：2026-05-31 13:06 CST

## 问题

用户反馈前端仍存在几处对普通使用者不友好的信息：

1. `Local LoRA 存在但未接入前端` 表达过于工程化，不清楚它与当前回答模型的关系。
2. 语料库列表只显示 `A- / 4 块 / 2 页`，没有说明权威等级含义，且证据块数量不应直接展示给用户。
3. 左上角文字“山”作为品牌图标不够精致。

## 已完成修复

1. 系统状态
   - 将 `Local LoRA` 改为 `训练模型`。
   - 将状态改为 `已存在，当前未用于前端回答`，更直接说明当前 Web UI 没有调用本地训练模型。
   - 将系统面板的英文工程标签改为中文：回答模型、服务提供方、向量模型、重排模型、证据库、训练模型。

2. 语料库
   - 移除文献行中的证据块数量。
   - 保留页数，因为页数对判断文献规模仍有直观价值。
   - 新增权威等级说明：
     - `A / A-`：优先引用
     - `B+ / B`：辅助核验
     - `C`：背景参考
   - 每条文献显示文献分类、权威等级说明和页数。

3. 品牌图标
   - 左上角从文字“山”替换为线性的山、水、日图标。
   - 图标仍使用现有配色和 44px 固定尺寸，避免影响侧栏布局。

4. 缓存版本
   - 静态资源版本号更新为 `20260531-1306`，确保浏览器加载新前端。

## 后续 UX 建议

1. 把“证据”栏改成可折叠的抽屉，在小屏幕上默认收起。
2. PDF 页图预览增加上一页/下一页按钮。
3. 对话输入框增加“正在检索/正在生成”的分阶段状态。
4. 语料库增加筛选条件：朝代、流派、技法、权威等级。
5. 来源列表可以显示更友好的短标题，并把完整文件名放到悬浮提示或详情里。
