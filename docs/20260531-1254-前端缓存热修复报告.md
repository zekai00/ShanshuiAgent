# 前端缓存热修复报告

生成时间：2026-05-31 12:54 CST

## 问题

用户打开 `http://127.0.0.1:7861/` 后看到的仍是旧前端，表现为此前提到的入口页、引用页图、回车发送、隐藏旧标签等改动没有生效。

## 排查

服务端实际返回内容已经包含新实现：

- 首页 HTML 已包含 `intro-screen`。
- `/assets/app.js` 已包含 `openPdfPreview`、Enter 发送、来源折叠和旧标签清理逻辑。
- `/assets/styles.css` 已包含入口页、PDF 弹窗和来源折叠样式。

因此更大概率是浏览器缓存旧的 `app.js` / `styles.css`，或页面是在服务重启前打开后未刷新。

## 修复

1. 给 `index.html` 中的静态资源增加版本参数：
   - `/assets/styles.css?v=20260531-1254`
   - `/assets/app.js?v=20260531-1254`

2. 给本地 Web 服务增加禁缓存响应头：
   - `Cache-Control: no-store, max-age=0`
   - `Pragma: no-cache`

禁缓存范围只覆盖首页和 `/assets/` 静态资源，避免本地开发时继续看到旧前端。

## 操作建议

服务重启后重新打开 `http://127.0.0.1:7861/`。如果浏览器仍显示旧页面，关闭旧标签页后重新打开；这次资源 URL 已变更，正常不需要手动强制刷新。
