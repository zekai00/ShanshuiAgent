# 配置与运行入口优化报告

生成时间：2026-05-31 03:29 CST

## 本轮目标

本轮优化先处理项目的基础工程问题：把路径、模型位置、服务端口、Neo4j、Milvus、ComfyUI、运行时 SQLite 等配置从业务代码中抽离，避免后续 RAG、评测、前端继续依赖单机绝对路径。

## 已完成改动

1. 新增统一配置模块 `src/config.py`
   - 统一管理项目根目录、数据目录、Milvus Lite DB、ColBERT 张量、运行时 SQLite、用户记忆库、模型路径、设备、DeepSeek、Neo4j、检索服务、ComfyUI 等配置。
   - 默认值兼容当前机器，但全部可通过 `.env` 覆盖。
   - 新增 `ensure_runtime_dirs()`，把运行产物写到 `data/runtime`、`generated_images` 等目录，不再写入源码目录。

2. 新增共享领域词表 `src/domain_terms.py`
   - 将检索和 Neo4j 图谱构建中重复维护的山水画领域词表、别名归一化词典集中到一个模块。
   - 修正了原先 `"弘仁": "渐江"` 被放在领域类型词典中的错误，现在领域类型为 `PERSON`，别名归一化仍映射到 `渐江`。

3. 重构核心运行入口
   - Agent 节点、LangGraph checkpoint、用户记忆、绘图工具、检索工具改为读取统一配置。
   - 检索服务、离线入库、Neo4j 全量构图、证据 store 迁移、评测脚本、训练数据脚本、实验脚本、旧 Gradio UI 均去除运行逻辑中的本机绝对路径。
   - 新增 `.env.example`，用于说明本地需要配置的密钥、模型路径、服务地址和运行目录。

4. 顺手修复的问题
   - `scripts/run_ingestion.py` 原先从 `Workspace.ChineseLandscape...` 导入 BGE 引擎，已改为标准 `src.retrieval.bge_m3_engine`。
   - Neo4j 图谱写入由字符串拼接 Cypher 改为参数化 Cypher，避免引号转义、文本截断异常和注入风险。
   - Neo4j 检索不再默认使用密码 `neo4j`；未配置 `NEO4J_PASSWORD` 时降级为无图谱检索模式。

## 验证结果

- 已执行：`python -m py_compile $(find src scripts ui -name '*.py' -not -path '*/__pycache__/*')`
- 结果：通过。
- 已扫描跟踪文件中的运行逻辑硬编码：不再出现 `/root/Workspace`、`/root/models`、`cuda:0`、`Workspace.ChineseLandscape`、源码目录 SQLite 路径等硬编码。

## 剩余风险

- `.env.example` 中仍保留当前机器的建议默认路径，这是示例配置，不是业务代码依赖。
- `src/agent/legacy/` 仍有本地未跟踪旧代码含硬编码，但该目录被 `.gitignore` 排除，不属于当前提交范围。
- 旧 Gradio UI 已能使用当前 Agent 图，但后续会优先新增现代化前端，不再把旧 UI 作为主要入口。
