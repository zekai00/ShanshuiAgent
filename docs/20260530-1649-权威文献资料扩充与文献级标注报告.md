## 文献级标注含义

`sources.jsonl` 是来源级清单，保留完整候选来源，包括失败候选。

`文献级标注清单.jsonl` 是项目可用清单，只收录完整可用文档。每条记录包含：

- `doc_id`
- `title`
- `author`
- `source_file`
- `raw_path`
- `category`
- `source_type`
- `curation_axis`
- `authority_level`
- `authority_weight`
- `rag_import_weight`
- `dynasties`
- `topics`
- `source_url`
- `landing_page`
- `sha256`
- `file_size_bytes`

`高权重导入队列.jsonl` 是后续 Milvus/RAG 导入顺序。它不是训练集，也不是评测集，而是告诉导入脚本：哪些文献应该优先入库，入库后检索排序或证据选择时可以给更高权重。

当前权重规则：

- `A`：`authority_weight = 1.25`
- `A-`：`authority_weight = 1.15`
- 古代画论原典：`rag_import_weight` 额外加 `0.08`
- 博物馆图录/论文集：`rag_import_weight` 额外加 `0.05`

## 下载稳定性处理

扫描 PDF 下载不稳定的主要原因是：文件体积大、来源端限速或中断、动态阅读器页面不直接返回 PDF、断点续传不完整、链接返回 HTML 而不是 PDF。

本次脚本的处理原则：

- 只把通过 `%PDF-` 和 `%%EOF` 检查的文件标记为完整。
- 每个完整文件记录 `sha256` 和 `file_size_bytes`。
- DPM 分页 PDF 先逐页校验，再本地合并。
- 公开网页文本生成的 PDF 明确标注为“文本整理版”。
- 失败候选留在 `sources.jsonl`，但不进入 `文献级标注清单.jsonl` 和 `高权重导入队列.jsonl`。
