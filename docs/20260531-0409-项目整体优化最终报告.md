# 项目整体优化最终报告

生成时间：2026-05-31 04:09 CST

## 总结

本轮优化围绕项目当前最大风险展开：路径配置分散、RAG 证据链路不稳定、检索评测口径不准、错误前提题检索较弱、researcher 回答缺少前提核验约束，以及缺少可视化对话入口。

已完成代码重构、索引重建、评测回归、回答提示词优化和现代前端页面。

## 已完成模块

1. 配置与运行入口统一
   - 新增 `src/config.py` 和 `.env.example`。
   - 将脚本、agent、retrieval、ingestion 中的硬编码路径迁移到统一配置。
   - Neo4j 不再使用默认密码；未配置密码时自动降级为无图谱检索。

2. RAG 证据链路重构
   - 新增 `scripts/retrieval/build_authority_evidence_store.py`。
   - 新增 `scripts/retrieval/build_milvus_from_evidence_store.py`。
   - 新 authority evidence store：135 篇可抽取文本、3617 页、7956 个 chunk。
   - 新 Milvus Lite authority 索引：`data/vector_store/milvus_authority.db`。
   - 检索返回稳定包含 `source_file`、`title`、`page_start/page_end`、`chunk_id`、`raw_chunk_text`。

3. 检索评测口径修正
   - `scripts/eval/run_retrieval_baseline.py` 支持 `--direct` 直接加载检索器。
   - 新增来源归一化，避免同一文献新旧文件名被误判。
   - 研究员 v2 复核集：任一来源命中 188/194（96.91%），全部来源命中 180/194（92.78%）。

4. 错误前提检索优化
   - 扩展 `src/domain_terms.py` 的画史词表与现代干扰词表。
   - `src/retrieval/online_retrieval.py` 增加错误前提 query rewrite、双路检索、标题/主题软先验。
   - 错误前提专项：任一纠错来源命中从 13/23（56.52%）提升到 16/23（69.57%），全部来源命中从 12/23（52.17%）提升到 14/23（60.87%）。

5. researcher 回答前提核验
   - 修改 `src/agent/prompts/researcher_prompt.yaml`。
   - 对“是否”类、时代错置、现代技术错置、人物/流派混淆题，要求先给“前提判断”，再给证据化解释和来源。

6. 现代前端
   - 新增 `scripts/run_web_app.py`。
   - 新增 `ui/modern/index.html`、`ui/modern/styles.css`、`ui/modern/app.js`。
   - 页面包含对话、证据、语料、系统 4 个视图。
   - 当前服务已启动：`http://127.0.0.1:7861`。

## 当前判断

项目最大问题原本确实在 RAG 证据链路，而不是 LangGraph 编排。现在证据链路已稳定到可继续迭代的状态。LangGraph 的主管-研究员-画师分工可以保留，短期不需要大改。

下一阶段最应该集中精力的是回答质量评测与 SFT 数据建设：用现有 evidence store 生成“前提判断、依据与解释、来源”的标准答案，让小模型或 researcher 模型学会稳定引用证据、明确纠错、不顺着错误前提编造。

## 运行说明

前端：

```bash
CUDA_VISIBLE_DEVICES=1 CL_MODEL_DEVICE=cuda:0 python scripts/run_web_app.py --host 127.0.0.1 --port 7861
```

重建 authority evidence store：

```bash
python scripts/retrieval/build_authority_evidence_store.py
```

重建 authority Milvus：

```bash
python scripts/retrieval/build_milvus_from_evidence_store.py --force --device cuda:1 --batch-size 16 --insert-batch-size 256
```
