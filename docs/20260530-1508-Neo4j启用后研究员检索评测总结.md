因此，Neo4j 已经接入成功，但当前图谱对检索指标的增益有限。主要瓶颈不在“Neo4j 是否启动”，而在图谱结构和召回融合策略。

## 判断

当前 Neo4j 图谱包含 40551 个节点、69777 条关系，标签为 `Chunk`、`Sentence`、`Entity`。它能作为实体邻接召回的补充，但目前仍是从 chunk 文本中抽取实体再连接 chunk，缺少更强的文献级结构、同义实体归并、朝代/人物/流派约束、错误前提纠错规则和全文检索索引。

这解释了为什么启用 Neo4j 后总体指标没有明显变化：Milvus + source-aware lexical + evidence store 已经覆盖了显式文献约束题，而错误前提题需要的是实体纠错和类型约束，不只是“实体相邻 chunk”。

## 下一步建议

1. 不要继续优先做 DPO/PPO/GRPO；当前瓶颈仍是检索与证据组织，不是回答模型偏好优化。
2. 先重构错误前提专项的评测标签：区分 `anachronism_only`、`entity_mismatch`、`category_error`、`unsupported_claim`，只有需要文献纠错的题才算纠错证据来源命中。
3. 改造 Neo4j 图谱：增加 `Document`、`Dynasty`、`Person`、`School`、`Technique`、`Work` 等节点，让错误前提能通过类型约束判断，例如“倪瓒-元代”“四王-清代”“Stable Diffusion-现代技术”。
4. 给 Neo4j 增加全文/关键词索引或独立 BM25，避免图谱只提供邻接召回。
5. 改 RRF 融合：错误前提题应优先召回“被问实体的正确信息”和“错误类别的正确信息”，而不是只按原问题整体语义召回。
