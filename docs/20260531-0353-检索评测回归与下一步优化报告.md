## 结论

普通研究员检索已经具备可用基础：主要题型中 factual_qa、artist_prompt、comparison 的任一来源命中接近或达到 100%，说明“文献级证据链路 + authority Milvus + evidence store 回填”的方向是正确的。

当前最大短板不是普通 RAG 链路，而是错误前提题。错误前提专项只有 56.52% 任一纠错来源命中，原因主要是问题中的现代技术词会污染向量检索，例如 Midjourney、Stable Diffusion、RGB、BIM、NFT、手机短视频、区块链等词把检索方向带离中国山水画的历史概念。

## 下一步优化判断

不建议现在直接进入 DPO、PPO 或 GRPO。检索层还没有稳定处理错误前提/时代错置问题，直接训练回答模型会把检索缺陷转移到模型行为上。

下一步应先做检索问题改写与错前提防护：

1. 增加错误前提 query rewrite：从问题中抽取“历史主体”和“错误现代谓词”，检索时优先用历史主体、画学概念、时代/流派词构造纠错查询。
2. 增加双路检索：一路保留原问题，一路使用去噪后的历史查询；最终融合时提高原典、馆藏、权威论文的权重。
3. 在 researcher 回答节点加入“先核验前提”的提示与结构化证据引用，避免根据错误前提顺着回答。
4. 重新评测错误前提专项，目标是任一纠错来源命中达到 75% 以上，再考虑回答模型 SFT。

## 对训练策略的影响

短期不需要 RL。当前需要的是证据链路和回答格式的工程修正，然后用 SFT 学会“基于证据回答、先纠错、标明来源”。只有当 SFT 后仍出现偏好型问题，例如回答太啰嗦、证据引用顺序不符合审美、拒答边界不稳定，再考虑 DPO。PPO/GRPO 暂不适合这个阶段。
