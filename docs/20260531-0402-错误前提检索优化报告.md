# 错误前提检索优化报告

生成时间：2026-05-31 04:02 CST

## 优化目标

上一轮评测显示，普通研究员检索已经稳定，但错误前提专项较弱：现代技术词、时代错置词会污染向量检索，使系统拿不到足够的纠错证据。

本轮目标不是训练模型，而是在检索层先解决“问题问错时应该检索什么”的问题。

## 代码改动

1. 扩展 `src/domain_terms.py` 的画史实体与概念词表，补充王维、荆浩、范宽、米芾、米友仁、赵孟頫、董其昌、文徵明、唐寅、仇英等人物，以及边角构图、点景建筑、桥梁意象、禅境、米氏云山、富春山居图等概念/作品。
2. 增加 `MODERN_NOISE_TERMS`，覆盖 Midjourney、Stable Diffusion、Transformer、RGB、BIM、NFT、区块链、云计算、手机短视频、打印机、虚拟现实、商业店铺、油漆、抽象表现主义等错误前提干扰词。
3. 在 `src/retrieval/online_retrieval.py` 中加入错误前提 query rewrite：对无书名号硬约束的“是否”类问题，生成一条去噪后的画史核验查询，与原查询双路检索。
4. 加入文献标题/主题软先验：当原问题去噪后与文献标题、主题高度匹配时，把这些文献的块加入 RRF 融合，但不作为硬约束。
5. 对错误前提类 query 默认返回至少 5 个证据块，因为这类题通常需要同时检索“正确时代/流派”和“被混淆对象”两类证据。

## 判断

这轮优化有效，但还没有到可以说“错误前提检索已解决”的程度。任一纠错来源命中从 56.52% 提升到 69.57%，说明 query rewrite 和标题/主题软先验方向正确；但距离 75% 目标仍有差距。

剩余失败主要不是 Milvus 或 evidence store 的问题，而是金标来源本身需要更明确的主题路由。例如“王翚是否开发采样器”更容易检索到王翚馆藏条目或相关论文，但专项金标要求命中“四王现象透视”；“赵孟頫 NFT”更容易检索到赵孟頫专题论文，而不是“元代山水画衍变浅析”。

## 下一步

下一步不建议继续堆检索启发式。更合理的是在回答层加入“前提核验模板”：先指出错误前提，再用当前检索到的权威证据解释正确史实；同时保留专项评测中的来源命中指标，但不要把它作为唯一成功标准。

训练策略上仍然是：先做 SFT，让 researcher 学会前提核验、证据引用和拒绝顺着错误前提编造。暂时不需要 PPO/GRPO；DPO 可以等 SFT 后有成对偏好样本时再考虑。
