## 页面结构

页面包含 4 个视图：

- 对话：与山水画研究员直接对话，右侧展示当前证据块。
- 证据：单独检索 RAG 证据，便于调试召回结果。
- 语料：查看 authority evidence store 的文献清单。
- 系统：查看 LLM 配置、evidence store 路径、文献数和证据块数。

前端使用原生 HTML/CSS/JS，不依赖 npm 构建。视觉上采用深色工作台、低透明玻璃面板、青绿色/金色/朱红色点缀，以及 canvas 动态粒子背景。

## 后端能力

FastAPI 提供以下接口：

- `GET /`：前端页面。
- `GET /health`：服务、LLM 与 evidence store 状态。
- `GET /api/corpus`：语料清单。
- `POST /api/retrieve`：直接检索证据。
- `POST /api/chat`：检索证据后调用 DeepSeek 生成回答；如果未配置 `DEEPSEEK_API_KEY`，自动降级为证据摘要。

首次调用 `/api/chat` 或 `/api/retrieve` 会加载 BGE-M3、BGE-Reranker 和 Milvus Lite，启动后第一次请求会较慢。当前服务进程已在 `127.0.0.1:7861` 运行。
