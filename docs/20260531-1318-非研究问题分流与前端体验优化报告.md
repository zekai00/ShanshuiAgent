# 非研究问题分流与前端体验优化报告

生成时间：2026-05-31 13:18 CST

## 背景

用户反馈两类问题：

1. 闲聊或通用问题不应强行检索山水画证据，例如“今天天气真不错”“什么是爱”。
2. 前端还需要进一步优化：PDF 翻页、证据栏折叠、语料筛选、生成状态、来源短标题和示例问题。

## 已完成改动

### 1. 非研究问题跳过 RAG

后端新增轻量问题分流：

- 命中山水画、画论、画史、技法、朝代、画家、流派等关键词时，进入 RAG。
- 未命中研究范围时，不调用 Milvus，不返回证据，不生成来源。

验证样例：

- `今天天气真不错`：返回直接闲聊回答，证据数为 0。
- `什么是爱`：返回边界说明和简短通用回答，证据数为 0。
- `青绿山水是什么？`：进入 RAG，返回 5 条证据。

### 2. PDF 页图翻页

证据 payload 增加 `page_count`。

PDF 预览弹窗新增：

- 上一页
- 下一页
- 当前页 / 总页数

翻页仍通过 `/api/pdf-page?source_file=...&page=...` 渲染，保留后端白名单校验。

### 3. 证据栏折叠

对话页新增“隐藏证据 / 显示证据”按钮。

- 桌面端默认显示证据栏。
- 小屏幕默认收起证据栏，减少阅读干扰。

### 4. 语料库筛选

`/api/corpus` 现在返回每篇文献的 facets：

- 朝代：`dynasties`
- 流派：`lineages_schools`
- 技法：`styles_techniques`
- 权威等级：`authority_level`

前端语料库页新增四个筛选器：

- 全部朝代
- 全部流派
- 全部技法
- 全部等级

### 5. 对话阶段状态

前端发送问题后显示更细阶段：

- 准备检索
- 检索中
- 重排中
- 生成中
- 直接回答

其中非研究问题会直接进入“直接回答”，不进入检索阶段。

### 6. 来源短标题

证据卡片优先显示文献标题或短标题。

完整文件名保存在 `title` 悬浮提示中，避免页面被冗长文件名占满。

模型提示词也已改为要求来源使用文献短标题和页码，不再鼓励输出冗长文件名。

### 7. 示例问题

对话页初始状态增加示例问题按钮：

- 青绿山水是什么？
- 宋代三远法
- 吴门画派特点
- 南北宗论影响

用户点击后可直接发送。

## LoRA 说明

当前已有训练产物主要是 Researcher 的 SFT LoRA。训练目标不是闲聊，而是让小模型学习：

1. 根据检索证据回答。
2. 证据不足时拒答。
3. 时代错置时纠正前提。
4. 不伪造引用。

它当前没有接入前端回答链路。前端仍使用 `deepseek-v4-flash` 生成回答。
